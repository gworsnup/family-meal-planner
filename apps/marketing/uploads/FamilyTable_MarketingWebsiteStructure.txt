

# **FamilyTable – Marketing Website Structure**

## **Overall Goal**

The marketing site should:

* Build trust and desirability
* Drive users to **start using the app immediately**

# **1. Home Page (Primary Conversion Page)**

## **Purpose**

* Instantly communicate value
* Show the product in action
* Drive sign-ups

## **Structure**

### **Hero Section**

* Headline:
  *“No more ‘what’s for dinner?’”*
* Subheadline:
  *“Collect recipes. Plan meals. Streamline shopping.”*
* CTA:
  **“Get Started”**
* Visual:

  * Scroll-driven animation (recipe → plan → shopping flow)

### **How It Works (3-Step System)**

A simple visual flow:

1. **Collect Recipes**
2. **Plan Meals**
3. **Shop Smarter**

Each step includes:

* Short description
* UI screenshot or animation

### **Product Deep Dive Sections**

* Recipe Import (from anywhere)

* Smart Shopping List

Focus on:

* Real UI

### **“Designed for Real Life” Section**

### **Social Proof / Future Placeholder**

### **Final CTA Section**

# **Design Direction**

## **Style**

* Clean, modern, minimal
* Premium but approachable
* Soft colours, lots of whitespace

---

## **Visual Strategy**

* UI-first (show product constantly)
* Subtle animations
* Scroll-based storytelling

---

## **Key Differentiator**

Unlike generic SaaS sites:

* Focus on **real-life usage**
* Show **entire workflow**, not isolated features
* Emphasise **simplicity and clarity**
